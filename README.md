# Конструктор сайта (Vanilla JS)

## Короткое описание

Интерактивный конструктор сайта с визуальным редактором, написанный на чистом JavaScript с использованием ООП. Позволяет динамически добавлять блоки контента (заголовки, текст, колонки, изображения) и сразу видеть результат.

## Ключевые моменты

- **Язык:** Vanilla JavaScript (ES6+)
- **Сборка:** Parcel (dev-сервер и бандлинг)
- **Архитектура:** ООП (класс `App` и модульная система блоков)
- **Интерфейс:** HTML + CSS

## Функционал

- Добавление блоков с заголовками
- Добавление текстовых блоков
- Добавление блоков с колонками
- Добавление изображений
- Очистка конструктора
- Панель управления с формами для добавления контента
- Визуальная область для предварительного просмотра

## Структура проекта

```
constructorVanilaJS/
├── src/
│   ├── index.html         # Главная HTML-страница
│   ├── index.js           # Точка входа (инициализация App)
│   ├── model.js           # Модель данных (начальная конфигурация)
│   ├── blocks/
│   │   ├── app.js         # Класс App (логика конструктора)
│   │   └── block.js       # Классы блоков (TitleBlock, TextBlock, ColumnsBlock, ImageBlock)
│   ├── image.js           # Утилиты для работы с изображениями
│   ├── utils.js           # Вспомогательные функции
│   ├── image.jpg          # Пример изображения
│   └── styles/
│       └── main.css       # Стили конструктора
├── package.json           # Зависимости и скрипты
└── dist/                  # Собранные файлы (генерируется Parcel)
```

## Зависимости

- **Parcel** (dev): для запуска локального dev-сервера с hot reload

Установка:

```bash
npm install
```

## Локальный запуск

Запустите dev-сервер с автоматическим открытием браузера:

```bash
npm run serve
```

Приложение откроется в браузере на `http://localhost:1234` (или другом порту, который Parcel укажет).

## Деплой

Проект автоматически деплоится на **GitHub Pages** при пуше в ветку `main` благодаря GitHub Actions.

**URL проекта:**  
[https://denisunderonov.github.io/constructorVanilaJS/](https://denisunderonov.github.io/constructorVanilaJS/)

### Настройка деплоя

1. В настройках репозитория (Settings → Pages) выберите источник: **GitHub Actions**
2. Workflow автоматически соберёт проект через Parcel и задеплоит папку `dist/` на GitHub Pages

## Технические детали

- **Модульная структура:** используются ES6-модули (`import/export`)
- **Паттерн:** каждый тип блока — отдельный класс (TitleBlock, TextBlock, ColumnsBlock, ImageBlock)
- **Рендеринг:** динамическое добавление HTML через `insertAdjacentHTML`
- **Модель:** начальная конфигурация блоков задается в `model.js`
- **Формы:** обработчики событий submit для добавления блоков
